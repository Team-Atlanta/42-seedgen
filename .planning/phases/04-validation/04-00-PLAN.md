```json

      "branches": {"covered": 150, "count": 1200},
      "functions": {"covered": 45, "count": 200}
    }
  }]
}
```

The validation scripts expect:
- Harness binary that can be executed with stdin input
- `.profraw` files generated by running harness
- `llvm-cov export` JSON format with data[0].totals structure
- Seeds directory with multiple seed files

seeds-coverage.json (improved coverage):
```json
{
  "data": [{
    "totals": {

        "count": 200,

  }]
}
```

This demonstrates:
- Branch improvement: 150 → 280 (+130 branches, +10.83%)
- Function improvement: 45 → 78 (+33 functions, +16.5%)
- Both metrics show improvement (satisfies VALD-02 assertion)

Structure matches exactly what llvm-cov export --format=text produces (verified from 04-RESEARCH.md examples).
